# Reference-Safe Rewrite Rules

Use these rules when running `$benchmark-copy-rewrite`.

## Inputs To Check

- 1-3 benchmark captions, scripts, screenshots, or templates
- user topic, product, audience, city, case, or personal story
- desired platform and format
- tone target: practical, friendly, sharp, emotional, or instructional
- must-keep facts and claims to avoid

## Workflow Rules

- Extract the structure before writing.
- Extract style without copying exact sentences.
- Build slots for audience, pain, scene, proof, method, result, and next action.
- Mark non-copyable elements.
- Fill only truthful user material into the slots.
- Run a final originality and claim check.

## Output Must Include

- template skeleton
- style parameters
- slot table
- non-copyable list
- 1-3 adapted drafts
- originality self-check

## Quality Checks

- The output must be usable from the material provided by the user.
- Recommendations must explain their reasoning.
- If evidence is weak or missing, mark it clearly.
- Keep the first result focused; do not expand into a full consulting report by default.
- Do not claim automatic publishing, guaranteed growth, guaranteed traffic, or guaranteed sales.
- Do not copy another creator's exact wording, images, story, identity, or claims.

## Public-Safe Wording

Prefer:

- workflow
- checklist
- review
- judgment
- public signal
- user-provided material
- next action

Avoid:

- private data extraction
- secret system
- guaranteed viral result
- guaranteed traffic growth
- copy-paste another creator's content
